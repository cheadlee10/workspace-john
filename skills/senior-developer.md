- Modular, testable code (unit + integration tests)
- Security-first (auth, encryption, input validation)
- Scalable design (databases, caching, queues)
- DRY (don't repeat yourself) — templates, abstractions
- Documentation-as-code (README, API docs)

2. Architecture design (tech choices, data model)
3. Iterative builds (MVP first, then features)
4. Testing & documentation
5. Deployment to production
6. Handoff with runbooks for maintenance
