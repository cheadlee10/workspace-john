# Perfect Outreach Email Playbook (Research-Synthesized)

## What actually improves opens + replies
1. **Subject line length:** 30–45 chars for mobile-first visibility.
2. **Subject style:** local personalization + clear benefit + light curiosity.
3. **Preview text:** complete the promise from subject (don’t repeat it).
4. **One CTA only:** a single binary ask (e.g., “Want me to send your live demo link?”).
5. **Scannable structure:** 2–3 short paragraphs + 3 bullets + one button.
6. **Trust signals:** local context, specific service proof, simple guarantee.

## Design rules for “professional + cutting edge”
- 600px centered container, table-based HTML for compatibility.
- Brand palette: Navy `#0F2340`, Gold `#D4A574`, Light bg `#F8F9FA`.
- Accessible contrast, 16px+ body text, 46px+ CTA tap target.
- Include hidden preheader text.
- Avoid heavy animations; use visual “movement” via gradient accents + layered cards.
- Dark-mode defensive styling: avoid pure black/white extremes, keep bordered modules.

## Subject line bank (high-intent local service)
1. `[City] leads are searching for [Business]`
2. `Quick win for [Business Name] this week`
3. `Built this for [Business Name] (live)`
4. `Your next 3 booked jobs from web`
5. `[FirstName], this page could close more calls`
6. `I mocked up a better site for [Business]`
7. `Stop losing mobile leads, [Business]`
8. `[City] homeowners can’t find you fast enough`
9. `Can I send your live homepage concept?`
10. `A cleaner web presence for [Business]`

## Preheader formulas
- `Fast, mobile-first, and built around your exact services.`
- `One page, one goal: more booked calls this month.`
- `I already drafted the layout and headline stack.`

## Copy skeleton (cold outreach)
- Line 1: Personal local observation.
- Line 2: Revenue gap from weak/no web presence.
- Line 3: Specific offer with speed + proof.
- Bullets: what they get (3 max).
- CTA: one question.

## Send QA gate (must pass before send)
- [ ] Subject < 45 chars
- [ ] Preheader set
- [ ] One CTA only
- [ ] Personalization tokens resolved
- [ ] Mobile test (320px)
- [ ] Dark mode sanity check
- [ ] Links + reply-to verified
- [ ] No spammy words (“guaranteed income”, “free!!!”, etc.)
