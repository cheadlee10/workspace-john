# Production SOP — Rapid Premium Site Delivery

## Phase 0: Qualification (30–45 min)
1. Complete intake template.
2. Confirm top 3 services + target city cluster.
3. Lock one primary CTA (call vs quote form vs scheduler).
4. Collect required asset pack.

Exit criteria:
- Intake complete >= 90%
- Asset pack meets minimum mandatory list

## Phase 1: Information Architecture (45–60 min)
1. Build sitemap:
   - Home
   - 3–6 service pages
   - 3–15 city pages (as available)
   - About
   - Reviews
   - Contact
2. Define internal linking hubs (home -> service -> city -> contact).
3. Assign schema type by page (LocalBusiness + Service + FAQ).

Exit criteria:
- Sitemap approved
- URL slug standard agreed (`/services/<service>`, `/locations/<city>`)

## Phase 2: Content Assembly (2–4 hrs)
1. Apply niche playbook messaging.
2. Draft home + top service pages first.
3. Build FAQ from real sales objections.
4. Produce unique city intros (avoid duplicate doorway content).

Exit criteria:
- Core pages complete
- Copy checked for local relevance + proof claims

## Phase 3: Design + Build (3–5 hrs)
1. Apply design system tokens.
2. Assemble page sections from section library.
3. Implement responsive behavior + sticky CTA mobile.
4. Add trust modules and conversion blocks.
5. Integrate forms/call links/event tracking.

Exit criteria:
- All templates render correctly at mobile/tablet/desktop
- CTA path tested end-to-end

## Phase 4: SEO + Performance (60–90 min)
1. Title/meta for all indexed pages.
2. H1/H2 structure audit.
3. Add LocalBusiness schema + FAQ schema where used.
4. Optimize images to WebP + srcset.
5. Generate sitemap.xml and robots.txt.
6. Check CWV baseline and fix major blockers.

Exit criteria:
- No broken links
- Lighthouse meets baseline thresholds

## Phase 5: QA Gate + Client Review (45–60 min)
1. Run `qa/qa-gates-checklist.md` pass.
2. Capture screenshots for signoff package.
3. Present prelaunch URL and collect edits in one round.

Exit criteria:
- All P0/P1 defects resolved
- Signoff template completed

## Phase 6: Launch + Post-Launch (30–45 min)
1. Point DNS and verify SSL.
2. Submit sitemap to Search Console/Bing.
3. Verify analytics, form submissions, call tracking.
4. Monitor 24h for errors/lead events.

Exit criteria:
- Live site reachable with valid SSL
- First test lead successfully received

## Production Speed Rules
- Reuse components; never redesign from scratch unless required.
- Batch page creation by template type.
- Timebox revision rounds (max 2 rounds in standard package).
- Maintain page inventory tracker with status per URL.
