# QA — Mobile WOW + Conversion Checklist (V2)

## A. Above-the-Fold Conversion
- [ ] Primary CTA visible without scroll on iPhone 12/13/14 viewport
- [ ] Phone CTA reachable within 1 tap from first screen
- [ ] Hero includes 3 trust elements (rating/license/response-time)
- [ ] No layout shift in first 3 seconds

## B. Mobile UX Polish
- [ ] Tap targets are >= 48px
- [ ] Sticky CTA does not overlap form submit or keyboard interaction
- [ ] Text remains readable at 320px width
- [ ] Service cards stack cleanly with no overflow

## C. Form Friction Control
- [ ] Required fields: name + phone only for quick capture variant
- [ ] Phone field opens numeric keypad
- [ ] Error states are inline and clear
- [ ] Submit success and fallback error messages are visible

## D. Performance
- [ ] Hero image webp <= 180KB
- [ ] Lighthouse Mobile Performance >= 88
- [ ] LCP < 2.3s / CLS < 0.08 / INP < 180ms
- [ ] Unused CSS/JS removed

## E. Trust + Compliance
- [ ] License/insurance claim has verifiable source or removed
- [ ] Warranty language is precise, not vague
- [ ] Service area list reflects actual coverage
- [ ] Phone and business name consistent across page/footer/schema
