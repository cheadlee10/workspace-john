## 6) Demo Flow Script (Simple, 3–5 Minutes)

### Demo Goal
Show a manual, error-prone workflow transformed into an automated, trackable process.

### Setup (before call)
- Prepare sample raw data file (`leads_raw.csv` or `ops_tasks.csv`)
- Open “Before” sheet (messy state)
- Open “After” dashboard (clean KPIs + status)
- Keep timer visible (optional for impact)

### Live Script
1. **Context (30 sec):**
   - “This is what your team does manually today: collect, clean, route, follow up, report.”
2. **Pain Snapshot (30 sec):**
   - Highlight duplicates, missing fields, inconsistent formatting, delayed follow-up.
3. **Run Automation (45 sec):**
   - Click one trigger/button (or run script).
   - Narrate: “Data is cleaned, deduped, owner assigned, and follow-up task created.”
4. **Show Output (60 sec):**
   - Clean table, owner routing, status pipeline, summary dashboard.
   - Point out alert/notification step.
5. **Business Impact (45 sec):**
   - “This replaces ~X manual steps and saves ~Y hours/week.”
   - “Faster response = more conversions, fewer dropped leads.”
6. **Close (30 sec):**
   - “We can deploy a version like this for your process in days. Want a free audit?”

### Demo Talk Track (word-for-word option)
“Most teams don’t have a lead problem—they have a workflow problem. Here’s raw intake data with duplicates, missing fields, and no clear owner. I run one automation, and now everything is standardized, assigned, and tracked. Follow-up tasks are generated instantly, and leadership gets a live dashboard without manual reporting. This is the difference between reacting and operating.”

### Objection Handling One-Liners
- **“We already have tools.”** Great—this improves how your tools work together.
- **“Sounds complex.”** We start with one high-ROI workflow and scale from there.
- **“Not sure on budget.”** We scope quick wins first so savings can fund phase two.
